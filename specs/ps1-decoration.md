# 仕様書: Shell PS1 デコレーションツール (highline)

## 概要

Powerline スタイルの Shell プロンプト（PS1）を生成するCLIツール。
シェルの PS1 設定から呼び出され、カレントパス・Git ブランチ・Kubernetes コンテキストなどの情報をセグメント形式で出力する。

## ユースケース

### UC-01: プロンプト文字列の生成

- **アクター**: シェル（bash / zsh）
- **事前条件**: `highline` バイナリが PATH に存在する
- **基本フロー**:
  1. シェルが PS1 評価時に `highline` を実行する
  2. ツールが各セグメントの情報を収集する
  3. セグメントを左から順に結合し、Powerline スタイルの文字列を標準出力に出力する
  4. シェルがその出力を PS1 として表示する
- **事後条件**: プロンプトにセグメントが表示される

### UC-02: セグメントの選択と順序設定

- **アクター**: ユーザー
- **基本フロー**:
  1. ユーザーがコマンドライン引数でセグメント名を指定する
  2. 指定された順序でセグメントを表示する
- **例**: `highline path git kube`
- **事後条件**: 指定セグメントのみが指定順序で表示される

### UC-03: セグメントが取得不能な場合のスキップ

- **アクター**: シェル
- **基本フロー**:
  1. セグメントの情報取得に失敗する（Git リポジトリ外、kubectl 未インストール等）
  2. そのセグメントを表示せずスキップする
  3. 他のセグメントは通常通り表示する
- **事後条件**: エラーにならず、有効なセグメントのみ表示される

---

## セグメント仕様

### path セグメント
- カレントディレクトリの絶対パスを表示する
- ホームディレクトリは `~` に短縮する
- 長いパスは中間を省略する（例: `~/dev/.../highline`）省略は設定可能

### git セグメント
- 現在の Git ブランチ名を表示する
- Git リポジトリ外では表示しない
- ワーキングツリーの状態を記号で付加する:
  - `*`: 変更あり（modified）
  - `+`: ステージ済み（staged）
  - `?`: 未追跡ファイルあり（untracked）

### kube セグメント
- `~/.kube/config` から現在のコンテキスト名を読み取り表示する
- ファイルが存在しない・コンテキスト未設定の場合は表示しない
- `KUBECONFIG` 環境変数が設定されている場合はそちらを優先する

## 出力仕様

### フォーマット

- セグメントは Powerline スタイルの区切り文字（``）でつなぐ
- 各セグメントは背景色・前景色付きの ANSI エスケープシーケンスで装飾する
- シェルのプロンプト幅計算を壊さないよう、ANSI エスケープは非表示文字として適切にラップする
  - bash: `\[...\]`
  - zsh: `%{...%}`
- シェル種別はコマンドライン引数 `--shell bash|zsh` で指定する（デフォルト: bash）

### カラーパレット（デフォルト）

| セグメント | 背景色       | 前景色  |
|-----------|-------------|--------|
| path      | Blue (4)    | White  |
| git       | Green (2)   | Black  |
| kube      | DarkCyan (6)| White  |

### コマンドライン引数

```
highline [--shell bash|zsh] [segment...]

  --shell   出力するシェル種別 (デフォルト: bash)
  segment   表示するセグメント名（省略時は path git kube の順で全表示）
```

## データ定義

### Segment インターフェース

```go
type Segment interface {
    Name() string
    Text() (string, error)  // 取得失敗時は error を返す（表示スキップ）
    Colors() (fg, bg Color)
}
```

### RenderedSegment

```go
type RenderedSegment struct {
    Text string
    FG   Color
    BG   Color
}
```

---

## 制約・非機能要件

- 実行時間: 200ms 以内（プロンプト表示の遅延を最小化）
- kubectl / git コマンドへの依存は避ける。ファイル・環境変数から直接読み取る
- 外部ライブラリの使用は最小限に抑える（標準ライブラリ優先）
- ANSI 256色は使わず、基本16色（ANSI 標準）を使用する

## 受け入れ条件

| # | 条件 |
|---|------|
| 1 | `highline path` を実行するとカレントパスが Powerline スタイルで出力される |
| 2 | `highline git` を Git リポジトリ内で実行するとブランチ名が表示される |
| 3 | `highline git` を Git リポジトリ外で実行しても何も出力されずエラーにならない |
| 4 | `highline kube` を実行すると現在の Kubernetes コンテキストが表示される |
| 5 | `highline path git kube` で3セグメントが結合して表示される |
| 6 | `--shell zsh` 指定時に zsh 用エスケープで出力される |
| 7 | `make build` でバイナリが生成される |
| 8 | `make test` で全テストが通る |
